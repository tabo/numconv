numconv — flexible numeric conversion for python¶

numconv¶

synopsys:Python library to convert strings to numbers and numbers to strings.

numconv converts a string into a number and a number into a string using default or user supplied encoding alphabets.

constants¶

numconv.BASE85¶
Alphabet defined in section 4 of RFC 1924. Supposed to be a joke (it is an April’s fools RFC after all), but is quite useful because can be used as a base for the most common numeric conversions.
numconv.BASE16¶
numconv.BASE32¶
numconv.BASE32HEX¶
numconv.BASE64¶
numconv.BASE64URL¶
Alphabets defined in RFC 4648. Not really for common numeric conversion use.

functions¶

numconv.int2str(num, radix=10, alphabet='0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz!#$%&()*+-;<=>?@^_`{|}~')¶

Converts an integer into a string.

Parameters:
  • num – A numeric value to be converted to another base as a string.
  • radix – The base that will be used in the conversion. The default value is 10 for decimal conversion.
  • alphabet –

    A string that will be used as a encoding alphabet.

    The length of the alphabet can be longer than the radix. In this case the alphabet will be internally truncated.

    The default value is numconv.BASE85
Return type:

string

when num isn’t an integer
Raises ValueError:
 

when num isn’t positive
Raises TypeError:
 

when radix isn’t an integer
Raises ValueError:
 

when radix is invalid
Raises ValueError:
 

when alphabet has duplicated characters

Examples (taken from tests.py):

3735928559 to hexadecimal:

>> numconv.int2str(3735928559, 16)
'DEADBEEF'

10284 to binary:

>> numconv.int2str(19284, 2)
'100101101010100'

37 to base 4 using a custom dictionary:

>> numconv.int2str(37, 4, 'rofl')
'foo'

Very large number to BASE85:

>> numconv.int2str(2693233728041137L, 85)
'~123AFz@'
numconv.str2int(num, radix=10, alphabet='0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz!#$%&()*+-;<=>?@^_`{|}~')¶

Converts a string into an integer.

If possible, the built-in python conversion will be used for speed porpuses.

Parameters:
  • num – A string that will be converted to an integer.
  • radix – The base that will be used in the conversion. The default value is 10 for decimal conversion.
  • alphabet –

    A string that will be used as a encoding alphabet.

    The length of the alphabet can be longer than the radix. In this case the alphabet will be internally truncated.

    The default value is numconv.BASE85
Return type:

integer
Raises TypeError:
 

when radix isn’t an integer
Raises ValueError:
 

when radix is invalid
Raises ValueError:
 

when num is invalid
Raises ValueError:
 

when alphabet has duplicated characters

Examples (taken from tests.py):

Hexadecimal ‘DEADBEEF’ to integer:

>> numconv.str2int('DEADBEEF', 16)
3735928559L

Binary ‘100101101010100’ to integer:

>> numconv.str2int('100101101010100', 2)
19284

Base 4 with custom encoding ‘foo’ to integer:

>> numconv.str2int('foo', 4, 'rofl')
37

BASE85 ‘~123AFz@’ to integer:

>> numconv.str2int('~123AFz@', 85)
2693233728041137L
